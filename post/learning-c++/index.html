第九讲: 类与对象

缺省构造函数，缺省拷贝函数，缺省析构函数，

  • 构造函数的初始化列表　在构造对象的时候同步构造内部对象而写在函数内部，是对象构造完之后，再执行.
  • 成员初始化顺序是按照成员定义的顺序
  • 析构函数只能一个，可以系统调用或自己调用。但自己调用时，只是删除数据，并没有释放内存

内联函数

  • 内联函数要定义在头文件里
  • 类内定义的函数，自动转为内联
  • 构造／析构函数，可能含有附加操作，慎用内联
  • inline是对编译器的建议
  • 常数据成员必须在初始化是定义，并且保持不变。
  • 常成员函数不修改对象成员的值，注意const标志 void GetElem(void * arg) const
  • 静态数据成员，由类所有对象共享 static unsigned int count; 且初始化必须在类外，且定义不能在头文件中
  • 静态成员函数，为访问静态数据成员 static int func(char* arg);　没有this指针。
  • singleton
  • free与delete相比不调用析构函数，直接将内存交给OS
  • 静态常数据成员，只能在类外部初始化
  • 慎用　友元函数与友元类，可以访问自己的私有数据，但自己未必能访问对方的private数据　friend 函数/类声明 写在类的定义里

继承

  • 继承与派生
    • 派生类拥有基类的所有属性与行为，只能添加，不能取消
  • 单继承　
    • class 派生类: [public/protected/private] 基类　{……};
    • 建议将访问频繁的基类成员定为protected p3.P::print();
    • 派生类的构造函数，基类的构造要在初始化列表里
  • 多继承 慎用　派生类中可能保护多个基类副本
    • class 派生类: [public/protected/private] 基类A,　[public/protected/private] 基类B {……};
  • 虚继承　
    • 取消多继承时派生类中公共基类的多个副本，只保留一份
    • 派生时使用virtual　　class 派生类: virtual [public/protected/private] 基类A,　[public/protected/private] 基类B {……};
  • 派生类的构造函数与析构函数
    • 构造函数，先执行基类的构造函数，按继承顺序；调用派生类新增成员的构造函数；调用派生类的构造函数
    • 析构函数顺序与构造函数完全相反
  • 类的赋值兼容性
    • 公有派生时，任何基类对象出现的地方，都可以用派生类取代。只有派生类中基类中部分生效
    • 保护派生和私有派生，不可以直接赋值

多态

  • 不同对象在接受相同消息，做出不同反应
  • 虚函数:　virtual 函数返回值　函数名(函数参数)
  • 被继承，也同样为virtual
    • 虚拟表　虚表
  • 纯虚函数
    • 充当占位函数，没有任何实现
    • 派生类负责其具体实现
    • virtual int func(int arg) = 0;
  • 抽象类
    • 带有纯虚函数的类
    • 虚构造函数

第十讲: 操作符重载

  • 四则运算符重载
  • 关系操作符重载
  • 下标操作符重载
  • 赋值操作符重载
    • 拷贝、赋值和析构三位一体，一般同时出现
    • 如果对象有指针成员，一般要重载这三个函数
    • 有时因为是浅拷贝，会出现系统崩溃
    • 移动语义(C++11)
      • A(A &&that)
      • 可以同时提供深copy和移动copy两套函数
      • 右值意义 -流操作符重载
    • cout支持重载，这点比printf优

第十一讲: 泛型编程

  • 泛型编程概览
    • 编写不依赖数据类型的代码，就是泛型编程
  • 异常处理机制
    • catch(int) catch(char) catch(Point) catch(…)
    • noexcept(true) noexcept(false) noexcept(noexcept(expr))
  • RTTI
    • #include typeid(e).name()
    • dynamic_cast
    • static_cast
      • 一般用于非类型式的普通数据对象转型
      • 不进行运行期型式检查，不安全; 若失败，结果无定义
    • reinterpret_cast
      • 重新解释位序列的意义
      • 由程序员保证正确
    • const_cast
      • 用与设置/取消量的const状态
  • 模板与型式参数化
    • 模板代码要放在头文件中，编译器需要看到模板源代码
    • 模板用途
      • 函数模板，类模板，元编程
